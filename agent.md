# agent.md

## Цель агента

Ты помогаешь Максиму создавать и поддерживать веб-приложение для личного AI-трекера трат.

Технологический стек проекта:

- Frontend: GitHub Pages
- Backend: Cloudflare Workers
- AI: OpenAI API
- Модели: GPT-4.1 mini / GPT-4.0 mini

Твоя роль:

- генерировать полный рабочий код для проекта;
- поддерживать текущую архитектуру без лишних отклонений;
- помогать расширять функциональность по запросу Максима;
- сохранять строгую совместимость со схемой данных расходов.

## Архитектура

### Frontend

Frontend работает как статический сайт на GitHub Pages.

Ограничения:

- без Vercel;
- без Node.js backend;
- без серверного рендеринга;
- только статические файлы (`index.html`, `style.css`, `app.js`).

Frontend должен:

- отображать форму добавления траты;
- отправлять запрос в Cloudflare Worker;
- сохранять нормализованные траты в `localStorage`;
- отображать таблицу расходов;
- строить диаграммы через Chart.js.

### Backend

Backend работает на Cloudflare Workers.

Worker должен:

- принимать новые траты через HTTP API;
- вызывать OpenAI API;
- нормализовать данные в строго заданную JSON-схему;
- сохранять расходы в JSON-массив и/или Cloudflare KV;
- возвращать готовые нормализованные данные фронтенду.

### Хранение данных

Данные о тратах хранятся:

- на фронтенде: в `localStorage`;
- на бэкенде: в Cloudflare Workers / Cloudflare KV.

Нельзя заменять эту схему хранения на другую без явного указания Максима.

## Схема данных

Каждая трата обязана иметь строго фиксированную структуру:

```json
{
  "id": 1,
  "date": "2026-04-24",
  "amount": 45.0,
  "quantity": 2,
  "currency": "UAH",
  "description_raw": "чипсы для Марка",
  "product_name": "чипсы",
  "category": "еда",
  "sub_category": "вкусняшки",
  "sub_sub_category": "чипсы",
  "for_whom": "friend",
  "notes": "взял две упаковки",
  "ai_hint": "это для Марка"
}
```

Запрещено:

- добавлять новые поля;
- переименовывать поля;
- менять типы полей;
- удалять поля;
- менять JSON-схему без явного указания Максима.

Поле `for_whom` может иметь только одно из следующих значений:

- `"myself"`
- `"friend"`
- `"girlfriend"`
- `"family"`
- `"pet"`
- `"gift"`
- `"loan"`
- `"household"`
- `"other"`

Если входное описание неоднозначно, нормализация всё равно должна возвращать одно из этих значений.

## Пользовательский флоу

На фронтенде есть форма с полями:

- `date`
- `amount`
- `quantity`
- `description_raw`
- `notes`
- `ai_hint`

### Поведение при сохранении

При нажатии на кнопку `Сохранить`:

1. фронтенд отправляет `POST` на:
   `https://your-app.your-worker.workers.dev/api/add-expense`
2. бэкенд нормализует данные через OpenAI API;
3. фронтенд получает нормализованный JSON-объект;
4. фронтенд сохраняет его в `localStorage`;
5. фронтенд обновляет таблицу и диаграммы.

### Получение списка трат

Endpoint:

`GET /api/expenses`

должен возвращать весь массив трат.

При генерации кода ты обязан:

- писать готовый рабочий код;
- не писать лишние объяснения, если Максим явно не просит объяснение;

- придерживаться минимальной и понятной структуры проекта;

Если Максим просит сгенерировать код, по умолчанию ты возвращаешь именно:

- `index.html`

- `app.js`
- `worker.js`
- пример промпта для OpenAI

Если задача касается только одной части проекта, возвращай код только для нужных файлов.

## Правила генерации решений

Всегда:

- ориентируйся на текущую архитектуру проекта;
- проверяй, чтобы код соответствовал фиксированной JSON-схеме;
- не придумывай новые сущности без необходимости;
- не усложняй проект лишней инфраструктурой;
- делай код максимально пригодным для прямой вставки в проект.

Если Максим просит расширить функциональность, ты можешь помогать с:

- фильтрами расходов;
- новыми диаграммами;
- группировками по категориям;
- аналитикой расходов;

- ре-нормализацией старых расходов.

## Ре-нормализация данных

Если требуется ре-нормализация:

- ты берёшь сырой JSON-массив расходов;
- повторно прогоняешь данные через `gpt-4.1 mini`;
- возвращаешь результат в виде `expenses_cleaned.json`.

При ре-нормализации ты обязан:

- сохранять ту же строгую схему объекта;
- не добавлять новые поля;
- не удалять существующие поля;
- обеспечивать корректные значения `category`, `sub_category`, `sub_sub_category`, `for_whom`;
- возвращать валидный JSON, пригодный для прямого использования в приложении.

## Ожидаемый формат ответа

По умолчанию:

- возвращай код или содержимое файлов в готовом виде;
- не добавляй лишние комментарии вне кода;
- не добавляй рассуждения, если Максим не просит их явно;
- если нужен prompt для OpenAI, возвращай его как отдельный готовый текстовый блок.

Если Максим просит обновить проект:

- сохраняй совместимость с существующими файлами;
- не ломай текущий флоу;
- не меняй схему расходов;
- вноси изменения точечно и аккуратно.
